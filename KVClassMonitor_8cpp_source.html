14 <h2>KVClassMonitor</h2>
15 <h4>track potential memory leaks</h4>
16 <p>
17 Simple tool to track which classes instance count increases over time.
18 Every time the Check() method is called we compare the instance count of every
19 class derived from TObject to the previous Check().
20 We print a warning message for every class whose instance count has increased.
21 </p>
22 <p>
23 YOU MUST ACTIVATE ROOT OBJECT TRACKING FOR THIS TO WORK!
24 In your .rootrc file, add/change this variable:<br>
25 Root.ObjectStat: 1
26 </p>
27 <p>
28 Note that, as the instance counts are stored in a TEnv, any time new classes are
29 instanciated, some class instances increase due to the TEnv itself changing.
30 Typical output in this case is:
31 </p>
32 <pre>
33 Info in <KVClassMonitor::Check>: Checking class instance statistics
34 Warning in <KVClassMonitor::Check>: TList increase from 4060 --> 4065
35 Warning in <KVClassMonitor::Check>: TEnvRec increase from 3032 --> 3038
36 Warning in <KVClassMonitor::Check>: TClass increase from 70 --> 71
37 Warning in <KVClassMonitor::Check>: TObjArray increase from 90 --> 92
38 </pre>
39 <p>
40 This is a singleton class: only one instance can exist at a time. A pointer to
41 the current instance can be retrieved by static method KVClassMonitor::GetInstance()
42 </p>
43 <!-- */
44 // --> END_HTML

151 void KVClassMonitor::Check()
152 {
153  // update class instance statistics
154  // print warning for every class whose number has increased
155  Info("Check", "Checking class instance statistics");
156  gObjectTable->UpdateInstCount();
157  TIter next(gROOT->GetListOfClasses());
158  TClass* cl;
159  while ((cl = (TClass*) next())) {
160  Int_t n = cl->GetInstanceCount();
161  Int_t old = fClassStats.GetValue(cl->GetName(), 0);
162  if (n != old) {
163  if (n > old) Warning("Check", "%s +%d (%d --> %d)", cl->GetName(), n - old, old, n);
164  else Warning("Check", "%s -%d (%d --> %d)", cl->GetName(), old - n, old, n);
165  }
166  fClassStats.SetValue(cl->GetName(), n);
167  }
168 }
